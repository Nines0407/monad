# Agentic Payment System - AI 智能体介绍

## 🎯 这是什么？
一个基于 **Monad 区块链的智能体原生安全支付系统**，使 AI 智能体能够在用户授权下自主执行链上支付，同时确保安全性、可控性和可审计性。

## 🏆 竞赛背景
- **赛事**: Monad Blitz 杭州 - Agentic Payment 挑战赛
- **目标**: 构建专为 AI 智能体设计的支付系统，而非人类
- **核心创新**: 从"人类钱包"范式转向"智能体钱包"范式

## 🔑 核心价值主张
- **智能体自主性**: AI 智能体可以自主支付资源费用（API、计算、存储）
- **用户控制**: 细粒度的权限管理和实时监督
- **企业级安全**: 多层安全的非托管架构
- **完整可审计性**: 每笔支付都可完全追踪和解释

## 🏗️ 架构亮点

### 智能合约层（Monad）
- ERC-4337 账户抽象钱包
- 会话密钥管理器（临时、受限密钥）
- 策略引擎（可配置规则引擎）
- 审计日志记录器（不可变的支付记录）

### 客户端层
- 本地密钥管理（密钥永不离开设备）
- 策略检查器（快速本地评估）
- 会话密钥生成器（有限、过期的密钥）
- 交易构造器（为 Monad 优化）

### 集成层
- MCP 服务器（AI 智能体的模型上下文协议）
- CLI 工具（自动化和脚本）
- TypeScript SDK（开发者友好的 API）
- WebSocket API（实时更新）

## 🛡️ 安全模型
- **非托管**: 用户持有私钥，智能体无法访问
- **会话密钥**: 带严格限制的临时密钥（金额、时间、收款人）
- **策略引擎**: 基于规则的授权（预算、白名单、类别）
- **多层验证**: 本地 → 链上 → 审计追踪

## 🤖 AI 智能体集成

### 智能体如何交互
1. **检测支付需求**（API 调用、计算资源、数据访问）
2. **请求支付权限**并附带上下文（任务 ID、原因、金额）
3. **接收授权**通过有限的会话密钥
4. **自主执行支付**
5. **获取确认**并继续任务

### 可用的 MCP 工具
- `request_payment`: 请求并执行支付
- `check_budget`: 检查剩余预算和限制
- `get_payment_status`: 获取交易状态
- `list_payments`: 查看支付历史
- `update_permissions`: 修改智能体权限

## 📋 5项必需要求（竞赛）

1. **去中心化**: 非托管、用户控制密钥、开源
2. **安全配置**: 限制、白名单、人工审批、紧急控制
3. **智能体原生设计**: 为 AI 智能体优化，而非人类
4. **可审计和可解释**: 带上下文的完整支付记录
5. **恢复和权限管理**: 密钥轮换、多智能体管理、恢复

## 🎮 演示场景

### 场景 1: 基本支付流程
AI 智能体需要 API 访问 → 请求 0.50 美元支付 → 策略检查 → 生成会话密钥 → 执行支付 → 记录审计 → 智能体继续任务

### 场景 2: 策略执行
智能体超过每日限额 → 需要人工审批 → 用户审查并批准 → 支付继续进行并覆盖策略 → 审计显示策略例外

### 场景 3: 安全事件
检测到异常 → 所有操作暂停 → 调查 → 撤销受损密钥 → 恢复安全操作 → 增强安全策略
