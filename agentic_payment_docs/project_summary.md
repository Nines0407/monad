# Agentic Payment System - 项目设计总结

## 📋 项目概述

**Agentic Payment System** 是一个基于 **Monad 区块链的智能体原生安全支付系统**，专为 AI 智能体设计。项目参加 **Monad Blitz 杭州竞赛 - Agentic Payment 挑战赛**，目标让 AI 智能体在用户授权下能自主执行链上支付，同时保证安全、可控和完全可审计。

### 核心价值主张
- **智能体自主性**: AI 智能体可以自主支付资源费用（API、计算、存储）
- **用户控制**: 细粒度的权限管理和实时监督
- **企业级安全**: 多层安全的非托管架构
- **完整可审计性**: 每笔支付都可完全追踪和解释

## 🎯 5项核心竞赛要求

### 1. 去中心化与非托管
- 用户自持私钥，AI 智能体永远无法接触真实私钥
- 基于 ERC-4337 账户抽象标准
- 通过会话密钥（Session Key）授予临时权限
- 跨平台支持：macOS/Linux/Windows，AMD/Intel/Apple M 架构
- 完全开源：MIT/Apache 2.0 双重许可证

### 2. 多层安全配置
- **限额管理**: 单笔交易上限、每日/每周预算限制、时间控制
- **白名单机制**: 允许的收款地址、智能合约、代币类型、已验证服务商
- **方法级限制**: 特定合约函数调用权限，禁止危险操作
- **人工审批**: 阈值触发的人工审批，支持 Telegram/App/浏览器插件通知
- **紧急控制**: 一键暂停所有权限，即时会话密钥撤销

### 3. 智能体原生设计
- **权限请求流程**: 智能体明确请求支付权限并说明理由和上下文
- **临时授权**: 基于任务的短期授权，任务完成后自动过期
- **支付上下文**: 与任务ID、智能体ID、用户目标关联的记录
- **失败处理**: 重试逻辑，失败时备用支付路径
- **集成接口**: MCP 服务器协议、CLI 工具、TypeScript SDK

### 4. 完整可审计性
- **结构化支付记录**: 任务ID、智能体ID、用户ID、收款地址、金额、资产类型等
- **策略决策跟踪**: 评估的策略、规则、违规详情、决策原因
- **审计事件日志**: 完整的系统事件时间线
- **数据导出**: 支持 CSV、JSON、PDF 格式导出
- **链上存储**: 重要元数据在 Monad 区块链上永久存储

### 5. 权限管理与恢复
- **钱包恢复**: 社交恢复（3-of-5）、硬件备份、多重签名备用
- **权限调整**: 实时更新、批量操作、条件触发
- **会话密钥轮换**: 自动计划轮换，泄露时应急响应
- **多智能体管理**: 基于角色的权限分配、配额分配
- **生命周期管理**: 入职、暂停、离职、归档全流程

## 🏗️ 系统架构

### 三层架构设计

#### 1. 智能合约层（Monad 链上）
- **AgentWallet.sol**: ERC-4337 账户抽象钱包主合约
- **SessionKeyManager.sol**: 会话密钥管理器（临时、受限密钥）
- **PolicyEngine.sol**: 策略引擎（可配置规则引擎）
- **AuditLogger.sol**: 审计日志记录器（不可变的支付记录）

#### 2. 客户端层（本地）
- **密钥管理**: 本地加密存储，密钥永不离开设备
- **会话密钥生成器**: 有限、过期的密钥生成
- **策略检查器**: 快速本地策略评估
- **交易构造器**: 为 Monad 优化的交易构建

#### 3. 集成层
- **MCP 服务器**: AI 智能体的模型上下文协议集成
- **CLI 工具**: 自动化和脚本支持
- **TypeScript SDK**: 开发者友好的 API
- **WebSocket API**: 实时更新和通知

### 核心数据流
```
AI 智能体支付请求 → 本地策略检查 → 会话密钥签名 → 
→ 交易构造 → Monad 网络提交 → 审计记录 → 结果返回
```

## 🎮 演示场景

### 场景 1: 基本支付流程
AI 智能体需要 API 访问 → 请求 0.50 美元支付 → 策略检查 → 生成会话密钥 → 执行支付 → 记录审计 → 智能体继续任务

### 场景 2: 策略执行
智能体超过每日限额 → 需要人工审批 → 用户审查并批准 → 支付继续进行并覆盖策略 → 审计显示策略例外

### 场景 3: 安全事件
检测到异常 → 所有操作暂停 → 调查 → 撤销受损密钥 → 恢复安全操作 → 增强安全策略
